# AGENTS

このファイルは、このリポジトリで作業するエージェント向けの常駐ガイドです。詳細な手順はここに増やさず、必要に応じて関連ドキュメントを参照してください。

## 中核原則

### 1. 推測で埋めず、事実確認を優先する

- 必要なファイル、設定、既存実装、Issue / PR の記述を確認してから判断すること。
- 根拠が確認できない内容は、事実として断定しないこと。
- 不明点や判断材料不足がある場合は、不明であることを明示すること。

### 2. 必要最小限の変更を選ぶ

- 依頼を満たすために必要な最小限の変更だけを行うこと。
- 根拠のない拡張、抽象化、将来対応、ついで修正を加えないこと。
- 変更理由を説明できない差分は作らないこと。

### 3. 変更範囲を局所化する

- 必要なファイル・必要な箇所だけを変更すること。
- 依頼範囲外の修正を混ぜないこと。
- 自分が作った一時ファイルや作業上の汚れだけを片付け、既存の未整理状態を勝手に触らないこと。

### 4. 成功基準を確認するまで完了扱いにしない

- 先に成功基準を Issue、依頼文、関連ドキュメントから確認すること。
- 変更後は、実行可能な確認手段で基準を満たしたか確認すること。
- 確認に失敗した場合は、基準を満たすまで修正と再確認を繰り返すこと。

## 主要ドキュメント

- `docs/workflow.md`: Issue / PR / label / branch / review / merge を含む開発ワークフローの詳細。
  具体的な進め方や運用ルールが必要な場合は、まずこのファイルを参照すること。

## 補足

- この `AGENTS.md` には高レベル原則と最小限の案内のみを置く。
- 詳細手順を追加したくなった場合は、まず既存ドキュメントへの追記や参照で解決できるかを検討する。
